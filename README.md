# Reflection: Learning Unit Testing and CI Pipelines

During a recent interview, I realized that I didn’t have much hands-on experience with **unit testing** and **CI/CD pipelines** like GitHub Actions (as well as promises in JavaScript). I wanted to gain some basic exposure , so I completely this small activity.

## What I Learned

1. **JavaScript Promises / Async-Await**  
   - Practiced creating and consuming Promises.  
   - Used `async/await` to handle asynchronous flows cleanly.

2. **Unit Testing with Jest**  
   - Wrote tests to validate small functions in isolation.  
   - Learned how tests catch regressions and help maintain reliable code.

3. **GitHub Actions / CI Pipeline**  
   - Set up a workflow to automatically run tests on every push.  
   - Learned the distinction between CI reporting failures and enforcing merges via branch protection rules.  
   - Debugged issues such as missing `package-lock.json` and Jest execution permissions.

## Reflection

This project helped me fill some gaps in my understanding and gave me a better glimpse into the world of testing and CI/CD. It was a fun experience debugging during the setup process!

Even though this was a small activity, I now feel more comfortable:
- Explaining how unit tests prevent bugs.  
- Demonstrating a basic CI pipeline.  
- Troubleshooting common pipeline issues in a real-world scenario.

I plan to continue building on this foundation by using what I've learned in future projects!
